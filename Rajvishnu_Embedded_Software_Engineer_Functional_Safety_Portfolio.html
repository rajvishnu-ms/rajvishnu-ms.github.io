DEDICATED HIL CASE STUDY · TDK SENSORS
Virtualised HiL platform for automotive sensor interface validation

A complete requirements to evidence workflow combining Vector CANoe, CANalyzer, VT System, CAPL, Python, virtual ECU and sensor models, interface state behaviour, fault stimulation, laboratory measurements, regression testing and release documentation.

Engineering challenge

Validate communication, timing, state transitions and fault behavior before every ECU, sensor and vehicle component was physically available.

My ownership

Derived test concepts, built the Vector environment, implemented CAPL models and automation, integrated physical stimulation, analysed failures and produced release evidence.

Why it is safety relevant

The platform verifies that a system detects abnormal behavior, enters the correct state, responds within the required time and recovers in a controlled way.

TEST SCOPE

LIN startup through header and wake-up pulse, sleep through inactivity and explicit sleep trigger, Task-Off behavior, timing, signal response and controlled recovery.

IMPLEMENTATION

CAPL-based state logic and stimulation in CANoe, with monitoring of message activity, transition timing, first valid response, inactive-state behavior and reactivation.

VERIFIED OUTCOME

The node entered the expected state within the required timing window, remained inactive when required and returned to normal communication only after a valid wake-up condition.
