Stimulus and control

At TDK, I generated CAN, LIN, SENT, SPI and OWI interface conditions through CAPL/Python-controlled scenarios. The tests covered startup, sleep, wake-up, timing deviation, invalid sensor values, missing nodes and fault recovery.

Data capture

Evidence came from bus traces, measurement channels, virtual model states, oscilloscope captures, signal-generator responses, DAQ data and structured configuration records.

Actionable pass/fail

Expected behaviour was compared with measured response, including timing, state transitions, error reporting, diagnostics and recovery. Failures were converted into reproducible regression cases.

CI/CD transfer

KIT work with Jenkins, Git and Docker, plus version-controlled ITK reports, supports the transfer into Ford-style bench scheduling, artifact handling, automated execution and traceable results.

Virtual ECU / BMS / sensor modelling

Inside Vector CANoe, I modelled virtual ECU communication and simulated BMS behaviour with internal CAPL sensor nodes. The virtual NBP8S pressure sensor model included register behaviour, timers, state-machine transitions, diagnostic responses and fault-injection capability.
